## ❗ Problemáticas / Necesidades
- Falta de control sobre el stock disponible.

- Dificultad para visualizar qué productos se usan con mayor frecuencia.

- Desorganización en la reposición de materiales.

- Ausencia de un sistema centralizado y digitalizado.

## 🧠 Fundamentación
La correcta administración del inventario es fundamental para el funcionamiento eficiente de cualquier organización. La carencia de herramientas digitales genera pérdidas de tiempo, errores humanos y falta de control. Este proyecto busca resolver esa necesidad mediante una aplicación que simplifique la gestión de materiales y brinde información útil para la toma de decisiones.

## 🎯 Objetivo General
Facilitar la administración de recursos y materiales dentro de una organización, permitiendo el control eficiente del inventario.

## 🎯 Objetivos Específicos
- Desarrollar una plataforma accesible para gestionar el stock de productos.

- Implementar alertas sobre niveles bajos de inventario.

- Generar reportes de productos más utilizados.

- Automatizar el seguimiento de reposiciones y consumos.

## 📊 Métricas
- El sistema permitirá medir y analizar:

- Niveles de stock bajo.

- Productos más utilizados.

- Frecuencia de pedidos de reposición.

## 🛠️ Primera Etapa del Desarrollo
Durante esta primera fase del proyecto, se trabajará en el maquetado inicial del sistema, el cual estará ubicado dentro de la carpeta mock. Esta carpeta incluirá la estructura base del sitio con los siguientes elementos:

- Assets/: Contendrá imágenes, íconos y recursos multimedia.

- Js/: Archivos JavaScript que manejarán funcionalidades básicas e interacciones.

- Css/: Hojas de estilo para el diseño visual de la interfaz.

- Html/: Archivos que representarán la estructura de las páginas del sistema.

Esta etapa se enfocará en diseñar una interfaz amigable y funcional, que sirva como base para futuras integraciones de lógica y conexión con bases de datos.
